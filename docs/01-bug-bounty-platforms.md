## 🏟️ Code4rena

### 概要
- 競技型の監査プラットフォーム
- "Wardens" と呼ばれるセキュリティリサーチャーが参加
- 発見された脆弱性の深刻度に応じて報酬を分配

### 特徴
- **1,607** 件のユニークな高深刻度脆弱性を発見
- **26,898** 件のユニークな発見
- **498** 件の監査を完了

### 最近のコンテスト例（2026年1月時点）
- **Rujira** - $40,000 プール
- **Fluid** - $93,500 プール

### 監査タイプ
1. **Competitive Audit** - 多数のWardenが参加する競技型
2. **Zenith** - トップ監査人による少人数制

### 主要クライアント
- EigenLayer
- Arbitrum
- Coinbase
- Chainlink
- Aave
- OpenSea

### 報酬構造
- 発見の深刻度と品質に基づく
- High/Medium/Low/QA カテゴリ
- 重複発見は報酬を分割

## 📈 バグバウンティ市場の動向

### トレンド
1. **報酬額の増加** - Critical脆弱性の報酬が上昇傾向
2. **競技型監査の人気** - Code4rena, Sherlock の成長
3. **AI統合** - 自動検出ツールとの連携
4. **マルチチェーン対応** - L2, サイドチェーンへの拡大

### 報酬例
| 脆弱性タイプ | 報酬レンジ |
|-------------|----------|
| Critical (資金喪失リスク) | $100K - $10M+ |
| High (重大な機能障害) | $10K - $100K |
| Medium (限定的影響) | $1K - $10K |
| Low (軽微な問題) | $100 - $1K |

## 🎯 バグハンターへのアドバイス

### 始め方
1. **スキルを磨く** - Solidity, DeFiプロトコルの理解
2. **CTFで練習** - Ethernaut, Damn Vulnerable DeFi
3. **小規模から始める** - 低〜中リスクの脆弱性を狙う
4. **レポートの質** - 明確で再現可能なPoCを書く

### 成功のポイント
- **ドメイン知識** - 特定のDeFiプロトコル（DEX, Lending等）に特化
- **新規プログラム** - 新しくローンチされたバウンティを狙う
- **コミュニティ参加** - Discord, Twitterで情報収集
- **継続的学習** - 最新の攻撃手法をフォロー
